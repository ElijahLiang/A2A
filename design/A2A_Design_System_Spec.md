# A2A Design System — Figma 规范文档
> 供 Lovart / Figma 设计师使用的完整设计系统说明
> 项目：A2A（Avatar To Avatar）— AI Agent 驱动的线下社交撮合平台

---

## 一、设计风格定义

### 核心视觉语言
- **风格**：像素艺术（Pixel Art）× 温暖纸质感（Warm Paper）
- **情感**：复古、温馨、轻松、有趣、略带神秘感
- **参照**：像素小镇游戏（如星露谷物语）+ 日式文具手账风格
- **主色调**：青绿色（Teal）为主，暖米白为底，金色点缀

## 三、字体系统（Typography）

### 字体家族

| 角色         | 字体名称                                              | 用途                    |
|-------------|------------------------------------------------------|------------------------|
| 像素字体（主）| `Fusion Pixel 12px Proportional` / `Zpix`            | 按钮、标题、标签         |
| 像素字体（备）| `Press Start 2P`                                     | 英文像素风备用           |
| 正文字体     | `Noto Sans SC`                                        | 中文正文、对话内容        |

### 字号比例（以 1px = 1pt 为基准）

| Token        | 大小    | 典型用途                |
|-------------|--------|------------------------|
| `text-xs`   | 7px    | 极小标签、角标            |
| `text-sm`   | 8px    | 辅助说明、角色等级         |
| `text-base` | 10px   | **按钮文字、弹窗标题**    |
| `text-md`   | 12px   | **正文内容、对话文字**    |
| `text-lg`   | 14px   | 次级标题                 |
| `text-xl`   | 16px   | **页面标题**             |
| `text-2xl`  | 20px   | 大标题、启动屏标语         |

### 字重
- 正常：400（Regular）
- 强调：700（Bold）
- 像素字体仅支持单一字重，视觉加粗通过描边（stroke）实现

## 五、边框与圆角（Border & Radius）

### 边框宽度
| Token          | 数值  | 用途             |
|---------------|------|-----------------|
| `border-thin`  | 1px  | 输入框、分割线    |
| `border-base`  | 2px  | **标准边框**（像素风）|
| `border-thick` | 3px  | 强调边框、active 状态 |

### 圆角
> 像素风格原则上使用**直角（0px）**或极小圆角，避免大圆角破坏像素感

| 场景       | 圆角值   |
|-----------|---------|
| 按钮       | 0px（直角）|
| 输入框     | 0px      |
| 对话气泡   | 2px      |
| 徽章/Badge | 1px      |
| 头像框     | 0px（像素风头像）|

## 六、阴影（Shadow）

像素风格阴影使用**硬阴影（Hard Shadow）**，即无模糊的偏移阴影：

| 名称         | 参数                                      | 用途         |
|-------------|------------------------------------------|-------------|
| 像素投影     | `3px 3px 0px rgba(17,21,19,0.35)`        | 按钮、卡片    |
| 轻投影       | `2px 2px 0px rgba(17,21,19,0.20)`        | 小元素       |
| 无投影       | `none`                                   | 悬停前默认   |
| 按钮按下     | `1px 1px 0px rgba(17,21,19,0.35)`（内缩）| 点击反馈     |

## 七、图标与图形（Icons & Graphics）

### 图标风格
- 使用**像素图标**（pixel icon），每个图标基于 16×16 或 32×32 像素网格绘制
- 颜色使用设计系统颜色，不使用渐变
- 线条宽度统一为 2px
- 禁止使用圆形图标（用正方形边框替代）

### 精灵（Sprite）规范
- 角色精灵：**64×64px**，分层设计（身体/头发/上衣/裤子/帽子）
- 建筑图标：统一尺寸，配色使用地图专用色
- 动画帧：以 4px 为基准的弹跳/行走动画

### 地图尺寸
- 地图画布：**1200×780px**（12列 × 10行，每格100×78px）
- 工具栏高度：**68px**
- 弹窗最大宽度：**720px**

## 九、动画规范（Animation）

> 所有动画遵循**像素感原则**：优先使用 steps() 步进，避免连续过渡

| 动画名称        | 时长    | 曲线           | 用途                  |
|----------------|--------|---------------|-----------------------|
| `pixel-bounce` | 400ms  | steps(4)      | 精灵跳跃              |
| `pixel-pulse`  | 800ms  | steps(2)      | 待机闪烁              |
| `avatar-hop`   | 300ms  | steps(3)      | 头像互动              |
| `dialog-pop`   | 150ms  | ease-out      | 弹窗出现              |
| `panel-rise`   | 250ms  | ease-out      | 面板从底部升起         |
| `bubble-pop`   | 200ms  | ease-out      | 对话气泡出现           |
| `fade-in`      | 200ms  | ease          | 通用淡入              |
| `cloud-drift`  | 20s    | linear infinite | 云朵漂移（背景装饰）  |
| `water-shimmer`| 2s     | ease infinite | 水面波光              |
| `door-open`    | 600ms  | ease-in-out   | 启动屏门打开          |
| `typewrite`    | 2s     | steps(20)     | 打字机效果（启动文字）|

### 动画时间基准
| Token           | 时长   | 用途              |
|----------------|-------|------------------|
| `dur-instant`  | 80ms  | 极速反馈          |
| `dur-fast`     | 150ms | 按钮 hover        |
| `dur-normal`   | 250ms | 弹窗、面板        |
| `dur-slow`     | 400ms | 页面过渡          |
| `dur-scene`    | 600ms | 场景切换          |

## 十一、页面流程（Page Flow）

```
启动屏（Landing）
  ↓ 点击门/任意键
登录页（Login）
  ↓ 手机号 + 验证码
人格测试（PersonaQuiz）— 5道MBTI题
  ↓ 完成测试，生成 AI 分身
像素小镇（TownMap）— 主界面
  ├── 点击建筑 → 场景弹窗（图书馆/咖啡馆/餐厅/广场）
  ├── 点击 Agent 精灵 → 信件对话框
  └── 邮局 → 查看匹配报告 & 确认见面
```

## 十三、给 Lovart 的补充说明

1. **所有边框使用硬边（无圆角）**，强化像素游戏质感
2. **投影使用硬阴影**（偏移+无模糊），而非高斯模糊投影
3. **字体优先使用像素字体**，正文内容使用 Noto Sans SC 保证中文可读性
4. **颜色不使用渐变**，使用纯色填充（地图地形除外）
5. **动画优先使用 steps() 步进**，模拟像素帧动画感觉
6. **响应式断点**：768px（移动端），900px（中等屏幕）
7. **Figma Variables** 建议使用 Collection 管理所有 Token，方便暗色模式扩展
